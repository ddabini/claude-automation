# DID 광고 프로그램 기획 브리프

## 프로젝트 개요
- **프로젝트명**: DID (Digital Information Display) 광고 프로그램
- **폴더 위치**: `클로드/03_DID광고프로그램/`
- **작성일**: 2026-02-25

---

## 목표
공간(오피스, 매장, 로비 등)에 설치된 4K 디스플레이에 광고 이미지·영상을 자동 롤링 재생하고, 관리자가 웹 브라우저에서 소재를 관리할 수 있는 시스템 구축.

---

## 핵심 구성요소

### 1. DID 디스플레이 페이지 (`display.html`)
- 해상도: 3,840 x 2,160 (4K UHD) 전체화면
- 이미지 및 영상 슬라이드쇼 자동 롤링
- 화면전환 효과: 페이드, 슬라이드(좌→우, 우→좌, 위→아래), 줌인/아웃
- 롤링 속도(각 소재 노출 시간) 설정값 반영
- 로고/브랜딩 오버레이 옵션
- 노출 설정된 소재만 재생

### 2. 어드민 관리자 페이지 (`admin.html`)
- **소재 업로드**: 이미지(JPG, PNG, GIF, WebP) + 영상(MP4, WebM)
- **소재 목록 관리**: 순서 변경(드래그앤드롭), 숨김/노출 토글
- **롤링 설정**: 각 소재 노출 시간(초) 개별 설정 / 전체 기본값 설정
- **화면전환 효과 선택**: 효과 종류 선택 + 전환 속도(ms) 설정
- **미리보기**: 현재 설정 상태 확인
- **데이터 내보내기/가져오기**: 설정값 JSON 백업·복원

---

## 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 구현 방식 | 단일 HTML x 2파일 | 설치 불필요, 브라우저에서 바로 실행 |
| 데이터 저장 | localStorage | 서버 없이 로컬에서 즉시 사용 가능 |
| 스타일 | CSS (인라인) | 외부 의존성 없음 |
| 소재 저장 | Base64 인코딩 → localStorage | 파일 서버 없이 브라우저에서 처리 |
| 영상 처리 | HTML5 video 태그 | 별도 플러그인 불필요 |

### localStorage 용량 한계 대응
- 브라우저 localStorage: 통상 5~10MB
- 고해상도 이미지/영상 업로드 시 용량 초과 가능
- 해결책: 파일 크기 경고 표시 + IndexedDB 폴백 처리 (더 큰 용량 지원)

## 화면 설계

### display.html
- 전체화면 검정 배경
- 소재 전환 애니메이션 (CSS transition)
- 영상 소재: autoplay + muted + loop=false (1회 재생 후 다음 소재로)
- 키보드 단축키: F → 전체화면 토글, Space → 일시정지/재생

### admin.html
- 라이트 테마 (사용자 선호)
- 사이드바 + 메인 컨텐츠 레이아웃
- 소재 카드 그리드 (썸네일 + 정보 + 컨트롤)
- 드래그앤드롭 순서 변경
- 실시간 설정 저장 (변경 즉시 localStorage 반영)

## 개발 우선순위

1. **display.html** — 4K 전체화면 슬라이드쇼 코어 기능
