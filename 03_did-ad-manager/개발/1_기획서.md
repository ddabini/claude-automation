# DID 광고 프로그램 기획서

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | DID (Digital Information Display) 광고 프로그램 |
| 목적 | 4K 디스플레이에 광고 이미지·영상 자동 롤링 + 웹 기반 관리자 페이지 |
| 작성일 | 2026-02-25 |

---

## 핵심 기능

### DID 디스플레이 (display.html)

| 기능 | 설명 |
|------|------|
| 4K 전체화면 | 3,840 x 2,160 기준 100vw × 100vh 꽉 채움 |
| 이미지 롤링 | 설정된 노출 시간(초) 후 다음 소재로 자동 전환 |
| 영상 재생 | 재생 완료 후 자동으로 다음 소재 |
| 전환 효과 | 페이드 / 슬라이드 좌·우·상 / 줌인 |
| 키보드 단축키 | F: 전체화면, Space: 일시정지, 방향키: 앞뒤 이동 |
| 커서 자동 숨김 | 3초 비활동 시 커서 숨김 |
| 실시간 동기화 | admin.html 변경 시 storage 이벤트로 자동 반영 |
| 소재 없음 안내 | 등록 소재 0개 시 안내 화면 표시 |

### 어드민 관리자 (admin.html)

| 기능 | 설명 |
|------|------|
| 파일 업로드 | 드래그앤드롭 + 클릭 업로드 |
| 지원 형식 | JPG, PNG, GIF, WebP, MP4, WebM |
| Base64 저장 | 파일을 텍스트로 변환하여 localStorage에 저장 |
| 순서 변경 | 위/아래 버튼으로 순서 조정 |
| 노출/숨김 | 소재별 토글 |
| 개별 노출시간 | 이미지별 노출시간 개별 설정 (null이면 기본값 사용) |
| 전환효과 설정 | 효과 종류 + 속도 슬라이더 |
| 백업/복원 | JSON 파일로 전체 데이터 내보내기/가져오기 |
| 저장공간 표시 | localStorage 사용량 실시간 표시 |

## 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 구현 방식 | 단일 HTML × 2파일 | 설치 없이 브라우저에서 바로 실행 |
| 스토리지 | localStorage | 서버 불필요, 즉시 사용 |
| 소재 저장 | Base64 인코딩 | 파일 서버 없이 브라우저에서 처리 |
| 스타일 | 인라인 CSS (CSS 변수 활용) | 외부 의존성 없음 |
| 영상 | HTML5 video | 별도 플러그인 불필요 |
| 외부 라이브러리 | 없음 | 순수 HTML/CSS/JS |

```json
// 설정 키: "did_settings"
{
  "defaultDuration": 5,
  "transition": "fade",
  "transitionSpeed": 800

// 소재 키: "did_media"
[
  {
    "id": "did-1234567890-abc123",
    "type": "image",
    "name": "banner01.jpg",
    "data": "data:image/jpeg;base64,...",
    "duration": null,
    "visible": true,
    "order": 0,
    "size": 524288,
    "uploadedAt": "2026-02-25T10:00:00.000Z"
  }
]
```

## 제약사항 및 한계

| 항목 | 내용 |
|------|------|
| localStorage 용량 | 브라우저별 5~10MB 제한 |
| 10MB 초과 파일 | 경고 표시 후 업로드 허용 (저장 실패 가능) |
| 크로스 탭 동기화 | 같은 브라우저의 다른 탭 간 storage 이벤트로 동기화 |
| 다른 기기 공유 | localStorage는 기기별 저장 → 공유 불가 (공유 필요 시 Supabase 업그레이드 필요) |
