# Pikbox — 기획서

## 1. 프로젝트 개요

**프로젝트명**: Pikbox
**목적**: 디자인 레퍼런스 이미지를 카테고리별로 수집·관리하고, AI로 유사 이미지를 자동 검색하는 개인 도구
**대상 사용자**: 디자이너, 기획자, 에이전트 시스템 운영자

## 2. 문제 정의

- 디자인 작업 시 레퍼런스를 여러 플랫폼에서 따로 수집해야 하는 번거로움
- 수집한 레퍼런스의 체계적 분류·관리 부재
- 에이전트 시스템(design-reference-archiver)과 수집 데이터 연동 부재

## 3. 핵심 기능

| 기능 | 설명 |
|------|------|
| 카테고리 관리 | 색상별 카테고리 생성·수정·삭제 |
| 이미지 업로드 | 드래그앤드롭 또는 파일 선택으로 이미지 업로드 |
| AI 분석 (Run) | Gemini API로 이미지 스타일·키워드 자동 추출 |
| 유사 검색 | Serper.dev로 Pinterest, Behance 등 플랫폼별 유사 이미지 검색 |
| 멀티선택 관리 | 여러 이미지 일괄 이동·삭제 |
| 에이전트 내보내기 | MD 파일로 내보내어 archiver 에이전트가 시드 데이터로 활용 |

## 4. 기술 스택

- **프론트엔드**: 단일 HTML (CSS/JS 인라인)
- **데이터베이스**: Supabase (REST API + Storage)
- **이미지 분석**: Gemini 1.5 Flash API
- **이미지 검색**: Serper.dev API (Google 이미지 검색 래퍼, 월 2,500회 무료)
- **폰트**: Noto Sans KR (Google Fonts CDN)

## 5. 필요한 API 키

| 키 | 용도 | 발급처 |
|----|------|--------|
| Supabase URL | DB/Storage 접속 주소 | supabase.com 프로젝트 설정 |
| Supabase Anon Key | 인증 (JWT 형식) | supabase.com 프로젝트 설정 |
| Gemini API Key | 이미지 분석 | aistudio.google.com |
| Serper API Key | 이미지 검색 | serper.dev |

## 6. 데이터 모델

### ref_categories (카테고리)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 기본키 |
| name | TEXT | 카테고리 이름 |
| description | TEXT | 설명 (선택) |
| color | TEXT | 색상 코드 |
| created_at | TIMESTAMPTZ | 생성 시각 |

### ref_images (이미지)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 기본키 |
| category_id | UUID | 소속 카테고리 (FK) |
| type | TEXT | 'uploaded' 또는 'collected' |
| title | TEXT | 이미지 제목 |
| source_url | TEXT | 원본 페이지 URL |
| image_url | TEXT | 이미지 파일 URL |
| platform | TEXT | 출처 플랫폼 |
| keywords | TEXT[] | 키워드 배열 |
| memo | TEXT | 사용자 메모 |
| created_at | TIMESTAMPTZ | 생성 시각 |

## 7. 에이전트 연동 방식

```
Pikbox → [에이전트 내보내기] → MD 파일 다운로드
→ 클로드/[프로젝트명]/레퍼런스/ 폴더에 저장
→ design-reference-archiver 에이전트가 시드 데이터로 활용
```

## 8. 프로젝트 일정

| 단계 | 내용 |
|------|------|
| Phase 1 | HTML 뼈대 + CSS 테마 + 설정 모달 |
| Phase 2 | 카테고리 CRUD + Supabase 연동 |
| Phase 3 | 이미지 업로드 + Storage + 그리드 |
| Phase 4 | Run 기능 (키워드 검색 + Serper.dev) |
| Phase 5 | 멀티선택 + 벌크 관리 |
| Phase 6 | 에이전트 내보내기 + archiver 연동 |
| Phase 7 | 문서화 |
