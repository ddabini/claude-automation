## 컨펌 이력 (학습형 컨펌 시스템)

### 컨펌 기록 #1
- 일시: 2026-02-25
- 프로젝트: 바로이집 풀케어 서비스 DID 광고
- 컨펌 단계: 에셋가이드 / v1~v3 반복
- 결과: v1 반려 → v2 반려 → v3 진행 중 (최종 피드백 대기)
- 반려 사유 (v1): 가로형으로 제작(세로형 요청), Noto Serif KR 사용(궁서체 느낌)
- 반려 사유 (v2): 서비스명 가독성 저하(gradient-clip), 텍스트 겹침, 디자인 요소 부족, 에이전트 미활용
- 학습 포인트:
  - 세리프/궁서체 계열 폰트 → 즉시 반려 대상
  - DID 광고 사이즈 요청 시 방향(가로/세로) 선확인 필수
  - gradient-clip 텍스트는 가독성 저하 → 솔리드 컬러 + text-shadow 사용
  - 레퍼런스/에셋 에이전트를 실질적으로 활용하지 않으면 사용자가 즉시 감지

## 팀장 판단 기준 (누적)

### DID 광고 요청 시 선제 확인 항목
1. 가로형(3840×2160) vs 세로형(2160×3840) — 반드시 선확인
2. 다크/라이트 테마 — DID는 다크 프리미엄 추천
3. 레퍼런스 에이전트 실질 활용 여부 확인

### 에이전트 파이프라인 준수 체크
- 레퍼런스 에이전트: 블로그 기사 나열 금지, 실제 디자인 이미지 URL 수집 필수
- 에셋 에이전트: 단순 목록이 아닌 실제 SVG 파일 생성 후 HTML에 적용 필수
- v1/v2 실패 원인: 파이프라인을 형식적으로만 따르고 실질 활용 미흡
