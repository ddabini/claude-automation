# LLM 앱 콘텐츠 규칙 (awesome-llm-apps 기반)

이 스킬은 Shubhamsaboo/awesome-llm-apps 저장소의 패턴을 바탕으로 LLM 앱을 설계하고 구현할 때의 규칙을 정의합니다.

## 저장소 참고

- GitHub: https://github.com/Shubhamsaboo/awesome-llm-apps
- 라이선스: Apache-2.0
- 지원 모델: OpenAI, Anthropic, Google Gemini, xAI, Llama, Qwen 등

---

## 앱 유형별 설계 규칙

### 1. 스타터 AI 에이전트 (입문 수준)

적합한 경우:
- 단일 목적의 간단한 작업 자동화
- 처음 LLM 앱을 만드는 경우

포함해야 할 것:
- 단일 에이전트 구조
- 명확한 입력/출력 정의
- requirements.txt 및 .env.example

예시 패턴: 블로그→팟캐스트 변환, 여행 계획, 데이터 분석

### 2. 고급 AI 에이전트

적합한 경우:
- 복잡한 다단계 추론이 필요한 경우
- 외부 도구(웹 검색, API)와 연동

포함해야 할 것:
- 도구(Tool) 정의 명시
- 에이전트 실행 루프 설명
- 에러 처리 및 재시도 로직

### 3. 멀티에이전트 팀

적합한 경우:
- 역할 분리가 필요한 복잡한 워크플로우
- 병렬 작업 처리

설계 원칙:
- 각 에이전트의 역할과 책임을 명확히 분리
- 에이전트 간 통신 방식 정의 (메시지 큐, 공유 메모리 등)
- 오케스트레이터 에이전트가 전체 흐름 관리

### 4. RAG (검색 증강 생성)

적합한 경우:
- 특정 문서/데이터에 기반한 답변이 필요한 경우
- 최신 정보 또는 비공개 데이터 활용

필수 구성 요소:
- 문서 로더 (Document Loader)
- 임베딩 모델 선택 및 명시
- 벡터 데이터베이스 (예: Pinecone, Chroma, Weaviate)
- 청킹(Chunking) 전략 정의

### 5. 메모리 기반 앱

포함해야 할 것:
- 단기 메모리: 대화 이력 (ConversationBufferMemory)
- 장기 메모리: 벡터 DB 또는 외부 저장소
- 메모리 초기화 및 업데이트 로직

### 6. MCP (Model Context Protocol) 에이전트

포함해야 할 것:
- MCP 서버 설정 파일
- 도구 정의 및 핸들러
- 클라이언트-서버 통신 방식

## 코드 구조 표준

프로젝트명/
├── README.md          # 프로젝트 설명, 실행 방법, 예시
├── requirements.txt   # 필요한 패키지 목록
├── .env.example       # 환경변수 템플릿 (절대 실제 키 포함 금지)
├── app.py             # 메인 앱 진입점
└── utils/             # 헬퍼 함수 모음 (선택)

## 모델 선택 기준

| 모델 | 적합한 상황 |
|------|-----------|
| GPT-4o | 복잡한 추론, 멀티모달 입력 |
| Claude 3.5+ | 긴 문서 처리, 안전성이 중요한 경우 |
| Gemini 1.5+ | 대용량 컨텍스트, Google 서비스 연동 |
| Llama / Qwen | 비용 절감, 로컬 실행, 데이터 프라이버시 |

## 필수 체크리스트

앱을 만들 때 반드시 확인:
- API 키가 코드에 하드코딩되어 있지 않은가?
- .env 파일이 .gitignore에 포함되어 있는가?
- 에러 처리가 적절히 되어 있는가?
- README에 실행 방법이 명확히 적혀 있는가?
- 모델 비용(토큰 사용량)을 고려했는가?
