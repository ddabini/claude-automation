# /build — 프로그램 개발 완전 자동화

사용자가 `/build`를 입력하면 기획부터 UI 설계, 코드 작성, README까지 자동으로 실행한다.
**모든 결과물은 단일 HTML 파일로 완성한다. 설치·로그인 없이 누구나 즉시 실행 가능.**

디자인이 포함된 프로젝트라면 **디자인 파이프라인의 에셋 적용 가이드를 반영**한다.

---

## 실행 순서

### Step 1. 브리프 수집
6가지만 물어본다:
```
1. 프로그램명
2. 이 프로그램이 해결하는 문제 (한 문장)
3. 핵심 기능 목록 (쉼표로 구분)
4. 주 사용자 (예: 나 혼자, 팀원, 고객)
5. UI 스타일 키워드 (예: 심플, 다크, 컬러풀 — 없으면 "심플 라이트")
6. 참고 디자인 또는 레퍼런스 URL (없으면 생략 가능)
```
→ 입력 즉시 자동 실행 시작.

### Step 2. 디자인 파이프라인 연계 (레퍼런스 URL 있을 때)

사용자가 레퍼런스 URL을 제공했거나 디자인 퀄리티가 중요한 프로젝트인 경우:

```
② design-reference-archiver — 시드 분석 + 유사 UI 레퍼런스 수집
   ↓ 디자인/구성 포인트 보고서
③ ★ 팀장 컨펌 ① ★ — 보고서 승인/수정
   ↓ 컨펌된 디자인 포인트
④ design-asset-generator — 메인폰트 선정 + 에셋 생성 + 에셋 적용 가이드
   ↓ 에셋 적용 가이드 (폰트·컬러·에셋 목록·레이아웃)
⑤ ★ 팀장 컨펌 ② ★ — 적용 가이드 승인/보완
```

→ 컨펌된 적용 가이드를 Step 3 코드 개발에 반영.
→ 레퍼런스 URL이 없고 단순 도구 개발이면 이 단계를 건너뛰고 바로 Step 3으로.

### Step 3. 기획 & UI/UX 설계 (자동)

**기획서 자동 작성:**
- 핵심 기능 Must Have / Nice to Have 구분
- 기술 스택 확정 (CDN 최소화 원칙)
- 데이터 저장 방식 결정 (localStorage / Firebase Realtime DB)

**UI/UX 설계 자동 작성:**
- ASCII 와이어프레임으로 화면 구성
- 컬러·폰트·컴포넌트 스펙
  - **에셋 적용 가이드가 있으면** → 가이드의 폰트·컬러·레이아웃을 그대로 적용
  - **가이드가 없으면** → 브리프 키워드 기반으로 자체 설계
- 초기·로딩·성공·에러 상태 처리 설계

### Step 4. 코드 개발 (자동)

**필수 준수 사항:**
```
✅ 단일 HTML 파일 — CSS·JS 전부 인라인
✅ 완성 코드 — TODO·placeholder 절대 금지
✅ 한국어 주석 — 모든 함수에 목적 설명 (비전공자도 이해 가능하게)
✅ 한국어 UI — 모든 텍스트·레이블·에러 메시지
✅ CSS 변수(:root) — 색상·폰트·간격 상단에 모아 관리
✅ 반응형 — 모바일(360px) · 태블릿(768px) · PC(1200px)
✅ 에러 처리 — alert 대신 인라인 메시지 박스
✅ 빈 상태(Empty State) — 데이터 없을 때 안내 화면
```

**에셋 적용 가이드 반영 (있을 경우):**
```
✅ 메인폰트 CDN <link> 삽입 + font-family 적용
✅ 컬러 팔레트를 CSS 변수에 반영 (--color-primary 등)
✅ 에셋 이미지를 지정 위치에 배치
✅ 레이아웃 가이드에 맞춘 구성
```

**데이터 처리 방식:**
- 단순 저장 (1대 기기) → localStorage
- 기기 간 공유 필요 → Firebase Realtime Database + Cloud Storage (Supabase 무료 플랜 소진)
- 파일 처리 → File API + FileReader
- 차트 → Chart.js CDN
- 엑셀 → SheetJS CDN

### Step 5. README 자동 작성
```
비전공자가 5분 안에 실행할 수 있는 수준으로 작성:
- 프로그램 한 줄 설명
- 주요 기능 목록
- 실행 방법 (단계별)
- 적용된 디자인 정보 (폰트·컬러 — 에셋 가이드 반영 시)
- 수정 가이드 (CSS 변수 위치 안내)
- 자주 묻는 질문 3가지
```

### Step 6. 저장 및 배포
```
✅ 개발 완료

📁 01_baroezip/[프로젝트명]/개발/ (바로이집) 또는 클로드/[프로젝트명]/개발/ (기타)
  ├── 1_기획서.md
  ├── 2_UIUX_설계서.md
  ├── 3_코드_산출물.md
  ├── README.md
  └── 파일/
      └── [프로그램명].html   ← 즉시 실행 가능

📁 [프로젝트명]/디자인/ (디자인 파이프라인 적용 시)
  ├── assets/          [에셋 파일]
  └── 적용가이드.md     [폰트·컬러·레이아웃]
```

→ 개발 완료 후 자동 흐름:
  1. `qa-tester` — 기능·반응형·보안·디자인 일치 자동 검증 → QA 보고서
  2. `deploy-manager` — QA 통과 시 Netlify/Firebase 자동 배포 → 배포 URL + 접속 확인

---

## 품질 기준
- 완성 즉시 브라우저에서 열어 실행 가능한 상태
- 기능 하나하나 실제 동작 (mock 데이터 아닌 실제 로직)
- 에러 상황에서도 페이지 리로드 없이 복구 가능
- 코드 300줄 이상이면 섹션별 주석 블록으로 구분
- Firebase 연동 시 config는 HTML 하드코딩 (프로젝트 규칙), 기타 API 키는 환경변수 또는 사용자 입력으로 처리
- **에셋 적용 가이드 반영 시**: 폰트·컬러·레이아웃이 가이드와 정확히 일치할 것
