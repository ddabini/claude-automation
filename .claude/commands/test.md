# /test — QA 품질 검증 자동화

사용자가 `/test`를 입력하면 지정된 파일(또는 가장 최근 개발 완료 파일)에 대해 QA 검증을 자동 실행한다.

---

## 실행 순서

### Step 1. 테스트 대상 확인
사용자에게 물어본다 (한 번에):
```
1. 테스트할 파일 경로 (없으면 가장 최근 개발 파일 자동 탐지)
2. 에셋 적용 가이드 경로 (없으면 생략 — 디자인 일관성 검증 스킵)
```
→ 입력 즉시 자동 실행 시작.

### Step 2. qa-tester 에이전트 호출
`qa-tester` 에이전트를 실행하여 7단계 검증 수행:

1. **테스트 대상 파악** — 파일 존재, 기획서/가이드 유무 확인
2. **코드 정적 분석** — HTML 구조, JS 문법, 미사용 변수, 보안 검사
3. **브라우저 동적 테스팅** — Playwright로 실제 렌더링·인터랙션 검증
4. **기능 검증** — 기획서 대비 기능 완성도 체크리스트
5. **디자인 일관성 검증** — 에셋 가이드 대비 폰트·컬러·레이아웃 일치 (가이드 있을 때)
6. **반응형 검증** — 360px / 768px / 1200px 각각 레이아웃 확인
7. **테스트 보고서 작성** — 심각도별 이슈 분류 + 수정안

### Step 3. 결과 보고
```
📋 QA 테스트 보고서
━━━━━━━━━━━━━━━━━━
■ 종합 결과: ✅ 통과 / ⚠️ 조건부 통과 / ❌ 재수정 필요

■ 심각도별 이슈
🔴 Critical (즉시 수정): N건
🟡 Major (수정 권장): N건
🟢 Minor (개선 사항): N건

■ 기능 완성도: N/M 항목 정상 (N%)
■ 반응형: 모바일 ✅ / 태블릿 ✅ / PC ✅
■ 권장 조치: [다음 단계 제안]
```

### Step 4. 자동 수정 루프 (Critical 발견 시)
```
qa-tester Critical 발견
  → leader에 보고 (이슈 목록 + 수정안)
  → program-development에 수정 지시
  → program-development 수정 완료
  → qa-tester 재검증
  → (최대 2회 루프, 2회 후에도 Critical → 사용자 에스컬레이션)
```

- **Minor/자동 수정 가능 이슈** → qa-tester가 즉시 직접 수정
  - 콘솔 에러(오타), CSS 변수 불일치, 미사용 변수, meta 태그 누락, console.log 제거
- **Major 이슈** → 수정안 제시 후 사용자 확인
- **Critical 이슈** → 자동 수정 루프 또는 사용자 에스컬레이션

### Step 5. 배포 연결 (선택)
- QA 통과 시: "배포를 진행할까요?" 확인
- 사용자 승인 → `deploy-manager` 에이전트 자동 연결

---

## 심각도 분류 기준

| 심각도 | 기준 | 예시 |
|--------|------|------|
| **Critical** | 핵심 기능 불가, 보안 취약점, 페이지 크래시 | 버튼 무반응, XSS 취약점, JS 에러로 멈춤 |
| **Major** | 기능 일부 오작동, 디자인 불일치, 반응형 깨짐 | 폼 초기화 안 됨, 모바일 레이아웃 깨짐 |
| **Minor** | 사소한 UI 문제, 개선 가능 사항 | 호버 효과 누락, 불필요한 스크롤 |

## 검증 대상 우선순위

1. **보안** — API 키 노출, XSS, eval() 사용 (가장 먼저)
2. **기능** — 모든 인터랙션 정상 동작
3. **반응형** — 3개 뷰포트 레이아웃
4. **디자인** — 에셋 가이드 일치 (있을 때)
5. **접근성** — 키보드 내비게이션, 색상 대비
