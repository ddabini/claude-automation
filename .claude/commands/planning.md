# /planning — 기획서 완전 자동화

사용자가 `/planning`을 입력하면 기획서 초안 작성부터 3종 첨삭, 최종 완성본까지 자동으로 실행한다.
**레퍼런스 퍼스트 원칙에 따라 레퍼런스 수집 후 기획서를 작성한다.**
중간에 멈추지 않는다. 모든 단계를 연속으로 실행해 완성 기획서를 한 번에 제출한다.

---

## 실행 순서

### Step 1. 브리프 수집
6가지만 물어본다:
```
1. 기획안 제목 또는 프로젝트명
2. 기획 목적 (무엇을 결정/승인받기 위한 것인가)
3. 보고 대상 (경영진·투자자·팀장 등)
4. 핵심 전략 또는 실행 방향 (간단하게)
5. 예산 규모와 타임라인 (없으면 "미정" 가능)
6. 참고 기획서 또는 레퍼런스 URL (없으면 생략 가능)
```
→ 입력 즉시 자동 실행 시작.

### Step 2. 리서치 + 레퍼런스 수집 (자동, 병렬)
두 에이전트를 **동시에** 호출합니다:

**`research-crawler`** (팩트 데이터):
- 시장 규모·성장률·트렌드 데이터 수집
- 유사 프로젝트 사례·벤치마킹 데이터
- 경쟁사 전략·성과 분석
- **출처 검증된 조사 보고서 제출** (기획안 근거 데이터로 직접 활용)

**`design-reference-archiver`** (시각적 구성 레퍼런스):
- 시드 URL이 있으면 → 해당 기획서/사이트의 **비주얼 레이아웃** 분석 + 유사 디자인 구성 확장 수집
- 시드 URL이 없으면 → 기획 주제·업종 키워드 기반 **시각적 구성·레이아웃** 레퍼런스 수집
- **구성 포인트 보고서 제출** (기획서의 시각적 구성·디자인 패턴 참고용)
- > 주의: 팩트 데이터(시장 규모, 경쟁사 분석 등)는 research-crawler 담당. design-reference-archiver는 비주얼 레퍼런스만 수집

### Step 3. 팀장 컨펌
두 보고서(조사 보고서 + 구성 포인트 보고서)를 사용자에게 통합 보고:
- 팩트 데이터가 기획 근거로 충분한지 확인
- 구성 레퍼런스 방향이 기획 의도와 맞는지 확인
- **승인** → 기획서 작성 시작 (research-crawler 데이터를 근거로 직접 인용)
- **수정 지시** → 해당 에이전트가 보완 후 재제출

### Step 4. 기획서 초안 작성 (자동)
레퍼런스에서 추출한 벤치마킹 인사이트를 반영하여 작성.
**구조: Why → What → How → So What**

```
Executive Summary        — 5초 안에 Yes 하게 만드는 압축 요약
1. 기획 배경 및 목적     — 현황·문제·기회의 창
2. 목표 설정             — 정량/정성 목표 (수치 포함)
3. 전략 방향             — 핵심 전략 3개 이하
4. 실행 계획             — 단계별 로드맵·타임라인
5. 자원 계획             — 예산·인력
6. 기대 효과 및 ROI      — 수치 중심
7. 리스크 및 대응        — 가능성·영향도·대응책
8. 의사결정 요청 사항    — 승인 항목·필요 자원·결정 기한
```

### Step 5. 3종 첨삭 자동 실행

**① 논리 구조 첨삭**
- Why-What-How 흐름 점검
- 논리 비약·빈틈 식별 및 개선안 제시
- 의사결정자 예상 반론 시뮬레이션

**② 설득력·임팩트 강화**
- Executive Summary 더 강력하게 재작성
- 손실 회피·사회적 증거·구체성 기법 적용
- 핵심 문장 3개 선정 후 강화 버전 제시
- (레퍼런스 사례의 성과 데이터를 설득 근거로 활용)

**③ 데이터·근거 보완**
- 근거 없는 주장 전부 식별
- 필요한 데이터 유형·확보 방법 제안
- 자체 데이터 생성 방안 (설문·파일럿·인터뷰) 제시
- (레퍼런스에서 확보한 시장 데이터·사례 수치 반영)

### Step 6. 최종 완성본 자동 생성
3종 첨삭을 전부 반영한 최종 기획서 작성.
- 데이터 미확보 항목: `[확인 필요: 어떤 데이터]` 표시
- 원본 구조 유지하면서 표현·논리·근거 전부 강화

### Step 7. 저장 및 보고
```
✅ 기획서 완료

📁 01_baroezip/[프로젝트명]/전략기획/ (바로이집) 또는 클로드/[프로젝트명]/전략기획/ (기타)
  ├── 1_기획안_초안.md
  ├── 2_첨삭_논리구조.md
  ├── 3_첨삭_설득력임팩트.md
  ├── 4_첨삭_데이터근거.md
  └── 5_기획안_최종완성본.md   ← 최종 제출용

📁 [프로젝트명]/레퍼런스/
  ├── README.md          ← 포인트 보고서
  └── links.md           ← 수집 링크 모음

핵심 개선 사항:
- [논리 보완 포인트]
- [설득력 강화 포인트]
- [추가 확보 필요 데이터]
```

## 품질 기준
- Executive Summary: 의사결정자가 5초 안에 판단 가능한 수준
- 모든 전략에 "왜 이 전략인가" 근거 포함
- 수치 없는 목표 금지 — 반드시 측정 가능한 형태로 작성
- 의사결정 요청 사항은 구체적 (승인 항목·금액·기한 명시)
- 레퍼런스에서 추출한 벤치마킹 인사이트가 기획서 곳곳에 반영되어 있을 것
