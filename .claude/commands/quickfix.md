# /quickfix — 경량 즉시 수정

사용자가 `/quickfix`를 입력하면 지정 파일의 사소한 이슈를 빠르게 자동 수정한다.
`/test`(전체 QA)보다 가볍고 빠른 즉시 수정 명령어.

---

## 실행 순서

### Step 1. 대상 확인
사용자에게 물어본다:
```
1. 수정할 파일 경로 (없으면 가장 최근 개발 파일 자동 탐지)
```
→ 입력 즉시 자동 실행 시작.

### Step 2. 정적 분석 + 즉시 수정
파일을 읽고 아래 항목을 자동으로 수정한다 (브라우저 실행 없이):

| # | 검사 항목 | 자동 수정 내용 |
|---|----------|--------------|
| 1 | `console.log` / `console.debug` | 제거 (console.error/warn은 유지) |
| 2 | 미사용 변수 | 제거 |
| 3 | CSS 변수 불일치 | 선언된 값으로 통일 |
| 4 | `<meta>` 태그 누락 | viewport, charset, description 자동 추가 |
| 5 | HTML 닫힘 태그 누락 | 자동 보완 |
| 6 | 하드코딩 API 키 (Firebase config 제외) | 경고 표시 (자동 수정 안 함) |
| 7 | `http://` mixed content | `https://`로 변환 |
| 8 | 불필요한 공백·빈 줄 | 정리 (3줄 이상 연속 빈 줄 → 1줄) |

### Step 3. 결과 보고
```
⚡ QuickFix 완료
━━━━━━━━━━━━━━━━━━
■ 수정 파일: [파일경로]
■ 자동 수정: N건
  - [수정1]: console.log 3건 제거
  - [수정2]: meta viewport 추가
  - [수정3]: ...
■ 경고 (수동 확인 필요): N건
  - [경고1]: API 키 하드코딩 감지 (line 42)

→ 전체 QA가 필요하면 `/test`를 실행하세요.
→ 배포하려면 `/deploy`를 실행하세요.
```

---

## /test와의 차이

| 항목 | `/quickfix` | `/test` |
|------|-------------|---------|
| 소요 시간 | ~30초 | 2~5분 |
| 브라우저 실행 | 없음 (정적 분석만) | Playwright 동적 테스트 |
| 반응형 검증 | 안 함 | 360px / 768px / 1200px |
| 기능 검증 | 안 함 | 인터랙션·이벤트 전체 |
| 자동 수정 | 즉시 수정 | 보고서 → 수정 루프 |
| 적합한 경우 | 배포 전 마지막 정리 | 개발 완료 후 전체 검증 |

---

## 수정 안전 규칙

- **원본 백업**: 수정 전 변경 사항을 사용자에게 보여주고 확인 후 적용
- **구조 변경 금지**: 기능 로직, 레이아웃 구조, 스타일 변경은 하지 않음
- **주석 보존**: 기존 한국어 주석은 절대 삭제하지 않음
- **확실한 것만 수정**: 판단이 애매한 항목은 경고로만 보고
