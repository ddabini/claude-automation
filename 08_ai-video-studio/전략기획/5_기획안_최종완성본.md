# VELA — AI 영상 제작·편집 플랫폼 기획서

> 버전: v1.0 (최종)
> 작성일: 2026-03-05
> 분류: 내부 기획 문서 (프로토타입)

---

## Executive Summary

**이 문서는 AI 기반 영상 제작·편집 플랫폼 "VELA"의 1차 구축 기획서입니다.**

---

**현재 상황**

영상 콘텐츠 제작에 5개 이상의 분산된 툴이 사용되고 있습니다.
AI 생성(Runway/Kling), 편집(Premiere/CapCut), 자막(Whisper 별도), BGM(Suno 별도)이
각각 다른 플랫폼에 올라가 있어 영상 1편당 반복적인 파일 변환·창 전환 비용이 발생합니다.

[확인 필요: 팀 내 영상 1편 실제 제작 소요 시간 측정]
[확인 필요: 현재 사용 중인 영상 툴 목록 + 월 구독료 합산]

**제안**

VELA는 위 5개 툴을 단일 플랫폼으로 통합합니다.

| 현재 | VELA |
|------|------|
| 5개 툴 + 파일 변환 반복 | 단일 플랫폼 원스톱 |
| 영상 1편당 2~4시간 예상 | 15~30분 목표 |
| 도구별 월 구독료 합산 | ~$38~53/월 (AI API + 인프라) |

**투자 규모 요약**

- 개발 기간: 4~6주 (Phase 1 MVP)
- 월 운영 비용: 약 5~7만 원
- 추가 인프라 셋업 비용: 없음 (기존 Firebase 인프라 재사용)

**결론**

Phase 1은 4~6주, 월 5~7만 원으로 영상 제작 생산성을 즉시 개선할 수 있는 투자입니다.
중장기적으로 Phase 3 로컬 AI 전환 시 외부 API 비용을 점진적으로 대체할 수 있습니다.

## 1. 왜 지금인가 (Why Now)

### 1-1. 현상 유지의 비용

지금 아무것도 하지 않을 경우 아래 비용이 계속 발생합니다.

**시간 비용 (반복 발생)**
- AI 생성 툴 → 편집 툴 파일 이동·변환: 작업마다 10~20분 낭비
- 4개 창을 오가며 작업하는 집중력 전환 비용
- [확인 필요: 팀 실측 기반 월간 누적 낭비 시간]

**도구 비용 (반복 지출)**
- [확인 필요: 현재 구독 중인 영상 툴 합산 비용]
- VELA Phase 1 운영 비용 ~$53/월과 직접 비교 가능

**기회 비용 (만들지 못한 영상)**
- 영상 제작 장벽으로 인해 착수하지 못한 마케팅·커뮤니케이션 영상 수
- SNS 쇼트폼 콘텐츠의 경쟁력은 발행 속도와 양에 달려 있음

### 1-2. 기회의 창이 열린 이유

1. **AI 영상 API 상업적 성숙** (2024~2025)
   - Runway Gen-3 Alpha, Kling AI 2.0, Stability SVD 등 외부 API 연동 가능 시점 도래
   - 2023년 이전에는 이 품질의 API가 존재하지 않았음

2. **기존 강자의 대응 지연**
   - Adobe Premiere, DaVinci Resolve: AI를 "추가 기능"으로 탑재 중
   - AI-First로 설계된 올인원 툴은 아직 파편화 상태
   - CapCut: 모바일 최적화, 데스크톱 전문 편집 기능 부족

3. **글로벌 AI 영상 시장 성장**
   - 2024년 기준 약 $5.1B → 2030년 $29.1B 예상 (CAGR 34.2%)
   - [확인 필요: Grand View Research 또는 MarketsandMarkets 최신 보고서]

4. **기존 인프라 재사용 가능**
   - Firebase Auth/Storage: did-ad-manager, Pikbox에서 이미 운영 중
   - 인프라 셋업 비용 없이 즉시 개발 착수 가능

## 3. 기능 설계 (Feature Spec)

### 3-1. Phase 1 — MVP 핵심 기능

#### A. 텍스트 → 영상 생성

**사용자 플로우**
```
[프롬프트 입력]
   ↓ 텍스트 입력 + 스타일 선택 (시네마틱/애니메이션/브이로그/광고/뉴스)
[AI 프롬프트 최적화] (GPT-4o)
   ↓ 입력 텍스트 → 최적화된 영상 생성 프롬프트로 변환
[생성 요청 + 진행률 표시]
   ↓ Runway 또는 Kling AI API 호출 (BullMQ 큐 비동기 처리)
   ↓ WebSocket으로 실시간 진행률 푸시 (예상 시간 표시)
[미리보기 + 액션]
   ↓ 브라우저 내 영상 재생
   → [MP4 다운로드] 또는 [편집 화면으로 이동]
```

**완료 기준 (DoD)**
- 프롬프트 입력 후 5분 이내에 영상 생성 완료
- 생성 중 WebSocket 실시간 진행률 표시
- MP4 (H.264) 포맷 다운로드
- 생성 실패 시 명확한 오류 메시지 + 재시도 기능
- API 폴백: Runway 실패 → Kling AI 자동 전환

#### B. 이미지 → 영상 변환

**사용자 플로우**
```
[이미지 업로드]
   ↓ 드래그앤드롭 / 파일 선택 (JPG/PNG/WebP, 최대 10MB)
[모션 스타일 선택]
   ↓ Zoom In / Zoom Out / Pan Left / Pan Right / Cinematic Float
[영상 길이 선택]
   ↓ 3초 / 5초 / 10초
[생성 & 다운로드]
```

**완료 기준 (DoD)**
- 이미지 업로드 후 5초 영상 생성 성공
- 5가지 이상 모션 스타일 선택 가능
- MP4 다운로드 가능

#### C. AI 자동 자막 생성

| 기능 | 상세 | 기술 |
|------|------|------|
| 음성 인식 | 영상 업로드 → 자동 자막 생성 | OpenAI Whisper API |
| 자막 편집 | 타이밍/내용 수정 가능한 에디터 | 자체 UI |
| 내보내기 | SRT 파일 다운로드 | 자체 구현 |
| 자막 병합 | 영상에 자막 하드 인코딩 | FFmpeg |

**성능 기준**: 한국어 음성 정확도 88~92% (배경 소음 없는 조건)
*근거: OpenAI Whisper large-v3 공식 벤치마크, 한국어 WER 8~12%*

**완료 기준 (DoD)**
- 5분 이하 영상 자막 생성 성공률 90% 이상 (음성 명확한 조건)
- SRT 파일 다운로드
- 자막 텍스트/타이밍 인라인 편집

#### D. 내보내기 (Export)

| 옵션 | 내용 |
|------|------|
| 포맷 | MP4 (H.264) — 범용 호환성 |
| 해상도 | 1080p / 720p |
| 비율 | 16:9 (가로) / 9:16 (세로, 쇼츠/릴스) / 1:1 (정방형) |
| 자막 | 없음 / 하드 인코딩 / SRT 별도 |
| 예상 시간 | 내보내기 전 예상 렌더 시간 표시 |

### 4-2. 데이터 흐름도 (텍스트→영상 핵심 루프)

```
사용자: 텍스트 입력 + 스타일 선택
        ↓
프론트엔드: GPT-4o로 프롬프트 최적화 요청
        ↓
백엔드: BullMQ에 영상 생성 Job 등록
        ↓ (비동기 처리)
워커: Runway Gen-3 Alpha API 호출
        ↓ (30초~3분 소요)
       실패 시 → Kling AI 자동 폴백
        ↓
워커: 생성 영상 Firebase Storage 업로드
        ↓
백엔드: Socket.io로 "완료" 이벤트 푸시
        ↓
프론트엔드: Firebase Storage URL로 영상 스트리밍 재생
        ↓
사용자: 다운로드 or 편집 화면으로 이동
```

### 4-3. AI API 통합 구조 (폴백 체인)

```
VideoGenerationService
├── RunwayProvider (1순위 — 고품질)
│   실패/타임아웃 시 →
├── KlingProvider (2순위 — 비용 효율)
│   실패 시 →
└── StabilityProvider (3순위 — 폴백)
    실패 시 → 사용자에게 오류 + 재시도 안내

공통 인터페이스:
  generate(prompt, style, duration) → Promise<{ jobId, estimatedTime }>
  getStatus(jobId) → Promise<{ progress, resultUrl }>
```

## 8. 리스크 & 대응 방안

### 8-1. 기술 리스크

| 리스크 | 심각도 | 대응 방안 |
|-------|-------|---------|
| AI API 응답 지연 (30초~5분) | 높음 | BullMQ 비동기 + WebSocket 진행률 + 예상 시간 표시 |
| AI API 서비스 중단/가격 급변 | 높음 | 멀티 프로바이더 폴백 구조 (Runway → Kling → Stability) |
| 브라우저 영상 처리 성능 한계 | 중간 | 경량 편집은 WebGL, 고품질 렌더는 서버 FFmpeg 분리 |
| Electron 앱 용량 (200MB+) | 낮음 | 로컬 AI 모델 선택적 다운로드, 핵심 기능은 클라우드 유지 |
| WebCodecs API 브라우저 호환성 | 중간 | 최신 Chrome/Edge 권장 + 구 버전은 FFmpeg.wasm 폴백 |

### 8-2. 비용 리스크 & 3중 통제

```
비용 폭증 방지 3중 안전장치:

1. 사용자별 일일 생성 한도 (예: 영상 5개/일)
2. 생성 전 예상 비용 고지 ("이 작업에 약 $0.35 사용됩니다")
3. 월별 예산 캡 설정 → 초과 시 자동 알림 + 생성 일시 중지

Phase 3 이후 로컬 AI 전환 시:
- Whisper.cpp: 자막 API 비용 $0로 전환
- rembg: 배경 제거 API 비용 $0로 전환
- 장기적 외부 API 의존도 감소 로드맵
```
