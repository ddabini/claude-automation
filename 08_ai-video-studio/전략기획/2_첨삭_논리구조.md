# 첨삭 1 — 논리 구조 강화

> 기준: Why → What → How 흐름의 비약·빈틈 제거

---

## 검토 결과 요약

| 항목 | 현재 상태 | 보완 필요 여부 |
|------|---------|-------------|
| Why (문제 인식) | 기회의 창이 명시되어 있으나 구체적 수치 부족 | 보완 필요 |
| What (목표·방향) | Phase 1~4 목표 설정 명확 | 양호 |
| How (실행 계획) | 개발 로드맵 주차별 상세 기술 | 양호 |
| Why→What 연결 | 문제→해결책의 인과관계 약함 | 보완 필요 |
| What→How 연결 | 목표와 로드맵 일치 | 양호 |
| 반론 대응 | 리스크 섹션 있으나 대응 구체성 부족 | 보완 필요 |

## 발견된 논리적 빈틈

### 빈틈 1: "Why Now" 부재

**문제**: 기획서가 "AI 영상 툴이 파편화되어 있어서 불편하다"는 수준에 머물고 있음.
왜 지금 만들어야 하는지 — 시장 타이밍, 기술 성숙도, 경쟁 창이 열린 이유 — 가 빠져 있음.

**추가할 내용**:
```
▶ Why Now: 기회의 창이 지금 열린 이유

1. AI 영상 생성 API가 2024~2025년에 상업적으로 성숙
   (Runway Gen-3, Kling AI 2.0, Sora API 출시 → 외부 API 연동 가능 시점)

2. 기존 강자(Adobe, DaVinci)의 AI 통합 지연
   → 전문가 도구는 AI를 "추가 기능"으로 탑재 중
   → AI-First로 설계된 툴은 아직 파편화 상태

3. 내부 수요 확인됨
   → 현재 팀 내 영상 작업에 5개 이상 툴 분산 사용 중
   → 워크플로우 통합 시 시간 절감 효과 즉시 발생

결론: 지금이 AI-First 영상 스튜디오를 내부 구축할 최적 타이밍
```

---

### 빈틈 2: MVP 범위 정의 모호

**문제**: Phase 1에서 "핵심 기능 3개"라고 했지만,
각 기능의 완성도 기준(Definition of Done)이 없어서 개발 완료 여부를 판단하기 어려움.

**추가할 내용**:
```
MVP 완료 기준 (DoD, Definition of Done)

텍스트→영상 생성:
  ✓ 프롬프트 입력 후 5분 이내에 영상 파일 생성 (최소 3초, 최대 10초)
  ✓ 생성 중 진행률 표시 (WebSocket 실시간)
  ✓ MP4 포맷으로 다운로드 가능
  ✓ 생성 실패 시 오류 메시지 + 재시도 가능

이미지→영상 변환:
  ✓ JPG/PNG 업로드 후 모션 영상(5초) 생성
  ✓ 3가지 이상 모션 스타일 선택 가능

자동 자막:
  ✓ 5분 이하 영상에 대해 자막 생성 성공률 90% 이상
  ✓ SRT 파일 다운로드 가능
```

---

### 빈틈 3: 기술 선택 근거 불균형

**문제**: 프론트엔드/백엔드 기술 선택 근거는 잘 정리되어 있으나,
인프라에서 Firebase를 선택한 이유가 "기존 경험"으로만 표현되어 있음.
Cloud Run을 백엔드 호스팅으로 선택한 이유도 빠져 있음.

**추가할 내용**:
```
인프라 선택 보강:

Firebase Storage 선택:
- 이미 did-ad-manager, pikbox에서 운영 중 → 설정 없이 즉시 재사용
- Firebase CDN 기본 내장 → 영상 전송 최적화 추가 설정 불필요
- 월 5GB 무료 스토리지 (프로토타입 단계에 충분)

Cloud Run 선택:
- 사용량 없을 때 비용 0 (서버리스 컨테이너)
- FFmpeg 도커 이미지로 영상 처리 환경 재현 가능
- Google 생태계에서 Firebase Storage 접근 시 지연 최소화
```

### 빈틈 4: Phase 간 전환 기준 없음

**문제**: Phase 1 → Phase 2로 언제 넘어가는지 기준이 없음.
"MVP 완성 후" 라는 막연한 기준은 우선순위 판단을 어렵게 함.

**추가할 내용**:
```
Phase 전환 기준 (Gate Criteria)

Phase 1 → Phase 2 전환 조건:
  ✓ MVP 3개 기능 모두 DoD 충족
  ✓ 내부 사용자 3명 이상이 주 1회 이상 사용
  ✓ 영상 생성 성공률 85% 이상 (API 오류 포함 전체 요청 기준)

Phase 2 → Phase 3 전환 조건:
  ✓ 타임라인 에디터에서 3개 이상 클립을 편집하고 내보내기 성공
  ✓ 월 활성 사용자 5명 이상 (내부)
  ✓ 핵심 편집 기능 NPS 8점 이상 (내부 평가)
```

### 빈틈 5: 내보내기(Export) 기능 상세 누락

**문제**: 영상 제작 플랫폼에서 내보내기는 핵심 기능임에도 기능 설계에서 누락됨.
해상도, 포맷, 비트레이트, 자막 병합 옵션 등이 없음.

**추가할 내용**:
```
내보내기 기능 상세 (MVP 포함):

지원 포맷: MP4 (H.264) — 범용 호환성 최우선
해상도 옵션:
  - 1080p (Full HD, 기본)
  - 720p (HD, 파일 크기 절감)
  - 4K는 Phase 2에서 추가
비율:
  - 16:9 (가로, 유튜브/PC)
  - 9:16 (세로, 릴스/쇼츠)
  - 1:1 (정방형, 인스타그램)
자막 처리:
  - 자막 없이 내보내기
  - 자막 하드 인코딩 (영상에 자막 직접 삽입)
  - SRT 파일만 별도 다운로드
예상 렌더 시간 표시: 내보내기 시작 전 예상 시간 고지
```

## 최종 논리 구조 평가

**강점**:
- Why-What-How 전체 틀 완비
- 기술 스택 선택 이유 비교 테이블 명확
- Phase별 로드맵 구체적

**보완 완료 후 기대 효과**:
- "왜 지금 만드는가"에 대한 설득력 향상
- 개발 완료 기준 명확화로 범위 크립(Scope Creep) 방지
- Phase 전환 게이트로 우선순위 판단 근거 제공
