## 개선 1: Executive Summary 재작성

**현재 버전의 문제점**:
- "AI 올인원 플랫폼"이라는 설명이 추상적
- 의사결정자가 "Yes"를 눌러야 하는 이유가 즉각 보이지 않음
- 숫자 없는 주장 ("시간 단축", "장벽 낮춤")

**개선된 Executive Summary**:

```
---
[ Executive Summary ]

지금 이 팀의 영상 제작 워크플로우에는 5개 툴이 쓰이고 있습니다.
각 툴 간 이동·포맷 변환·자막 처리에 영상 1개당 평균 2~4시간이 소요됩니다.
[확인 필요: 팀 내 실제 영상 제작 소요 시간 측정]

VELA는 이 5개 툴을 1개 플랫폼으로 통합합니다.
텍스트 입력 → 영상 생성 → 자막 → BGM → 내보내기까지
예상 소요 시간: 15~30분.

투자 규모:
  - 개발 기간: 4~6주 (Phase 1 MVP)
  - 월 운영 비용: 약 5~7만 원 (AI API + 인프라)

기대 효과:
  - 영상 1편 제작 시간: 현재 대비 70~90% 단축
  - 외부 편집 툴 구독료 절감 가능 (연간 [확인 필요: 현재 사용 중인 툴 비용])
  - 영상 콘텐츠 생산량: 현재 대비 3~5배 증가 가능

결론: Phase 1 기준 4~6주 개발 후 즉시 회수 가능한 생산성 투자입니다.
---
```

## 개선 2: 손실 회피 프레이밍 적용

**현재**: "이 기능을 만들면 이런 이점이 있습니다" (긍정적 프레임)
**개선**: "지금 안 만들면 이 비용이 계속 발생합니다" (손실 회피 프레임)

**적용 예시**:

```
▶ 현상 유지의 비용 (Status Quo Cost)

현재 워크플로우를 유지할 경우:

1. 시간 비용 (반복 발생)
   - 영상 툴 A → 툴 B 파일 변환: 매번 10~20분 낭비
   - AI 생성 → 편집 → 자막 → BGM을 4개 창에서 반복 전환: 집중력 손실
   - 예상 월간 누적 낭비 시간: [확인 필요: 팀 실측치]

2. 비용 비용 (반복 지출)
   - 현재 구독 중인 영상 툴 합산 비용: [확인 필요]
   - VELA Phase 1 운영 비용(~$53/월)과 단순 비교 시 [확인 필요]

3. 기회 비용 (발생하지 못한 영상)
   - 영상 제작 장벽으로 인해 만들지 못한 콘텐츠가 있는가?
   - SNS 콘텐츠, 마케팅 영상, 내부 커뮤니케이션 영상 기회 손실
```

## 개선 4: 의사결정자 예상 반론 대응

의사결정자가 가질 반론 3가지와 선제적 대응:

### 반론 1: "Runway, Pika 같은 전문 툴이 있는데 굳이 직접 만들어야 하나?"

**대응**:
```
외부 SaaS 구독 vs 내부 구축 비교:

외부 SaaS 구독 방식:
- Runway Pro: $35/월 (125 크레딧, 약 영상 25개)
- 편집 툴 + 자막 + BGM까지 합산: ~$80~150/월
- 데이터 외부 저장 (보안 통제 어려움)
- 우리 워크플로우에 맞게 커스터마이징 불가

VELA 내부 구축:
- 월 운영 비용: ~$38~53 (동일 사용량 기준)
- 내부 워크플로우 완전 커스터마이징 가능
- Phase 3부터 로컬 AI로 외부 API 비용 점진적 대체 가능
- 이미 구축된 Firebase 인프라 재사용으로 초기 비용 절감

결론: 중장기적으로 비용 우위 + 워크플로우 최적화 효과 확실
```

### 반론 2: "AI API 비용이 나중에 폭증하면?"

**대응**:
```
비용 통제 3중 안전장치:

1. 사용자별 일일 생성 한도 설정 (예: 영상 5개/일)
2. 생성 전 비용 예상액 표시 ("이 작업에 약 $0.35 사용됩니다")
3. 월별 예산 캡 설정 → 초과 시 자동 알림 + 생성 일시 중지

Phase 3 로컬 AI 전환 시 주요 API 비용 0으로 전환 가능:
- Whisper.cpp (로컬): 자막 비용 $0
- rembg (로컬): 배경 제거 비용 $0
```

### 반론 3: "4~6주 안에 정말 완성 가능한가?"

**대응**:
```
Phase 1 범위 제한의 근거:

이미 검증된 인프라 재사용:
- Firebase Auth/Storage: did-ad-manager, pikbox에서 운영 중
- BullMQ 패턴: 표준 Node.js 비동기 처리 패턴
- React + Vite: 팀 내 경험 기술 스택

신규 개발 필요 범위:
- AI API 프록시: 2~3일 (표준 REST 클라이언트)
- 영상 생성 UI: 3~5일 (기본 폼 + 진행률 바)
- Whisper 자막: 2~3일

총 신규 개발: 약 3~4주 / 버퍼 포함 4~6주
핵심 리스크: AI API 연동 불안정 → 폴백 API 미리 세팅으로 대응
```

## 개선 5: 섹션 우선순위 재배치 제안

현재 기획서 섹션 순서 → 의사결정자 관점 재배치:

```

2. 기능 설계
3. 기술 아키텍처
4. UI/UX
5. 기술 스택
6. 로드맵
7. 리스크
8. 비용

최종 문서 권장 순서 (의사결정자 최우선 정보 배치):
1. Executive Summary (1페이지 요약)
2. 현상 유지의 비용 (Why Now)
3. VELA 핵심 가치 제안
4. MVP 기능 & 완료 기준 (What)
5. 개발 로드맵 + Phase 전환 기준 (How)
6. 비용 추정 (투자 규모)
7. 리스크 & 대응 방안
8. 기술 아키텍처 (기술팀 참고용 — 부록으로 이동 가능)
```
