> 작성일: 2026-03-05
> 작성자: Strategy Planning Agent
> 버전: v0.1 (초안)

## Executive Summary

AI Video Studio는 **텍스트 한 줄에서 완성된 영상까지**, 비전문가도 프로급 영상을 제작할 수 있는 AI 올인원 영상 제작·편집 플랫폼입니다.

영상 제작의 진입 장벽은 여전히 높습니다. 전문 편집 소프트웨어(Adobe Premiere, DaVinci Resolve)의 학습 곡선, AI 영상 툴의 파편화(생성 따로·편집 따로·자막 따로), 클라우드 렌더링 비용 — 이 세 가지 문제를 하나의 플랫폼에서 해결합니다.

**1차 목표**: MVP 웹 앱 (텍스트→영상, 이미지→영상, AI 자막)
**최종 비전**: Electron 기반 로컬 AI 탑재 올인원 데스크톱 스튜디오

## 1. 프로젝트 비전 & 목표

### 1-1. 프로젝트명

| 구분 | 이름 |
|------|------|
| 브랜드명 (한국어) | AI 비디오 스튜디오 |
| 브랜드명 (영문) | **VELA** (Video + Elevate + AI) |
| 슬로건 | "텍스트에서 스크린으로, 생각이 영상이 되다" |
| 부제 | AI-Powered Video Creation & Editing Platform |

**네이밍 근거**: VELA는 항해 별자리(Vela Constellation) 이름이기도 하며 "나아간다"는 방향성을 내포. Video(V) + Elevate(EL) + AI(A)의 합성어로 AI가 영상 제작 수준을 끌어올린다는 메시지 전달.

---

### 1-2. 핵심 가치 제안 (Value Proposition)

```
VELA = 아이디어 → 완성 영상까지 걸리는 시간을 90% 단축
```

| 기존 방식 | VELA 방식 |
|---------|---------|
| 영상 기획 → 촬영 → 편집 → 자막 → 렌더링 (최소 수 시간) | 텍스트 입력 → AI 생성 → 보정 → 내보내기 (15~30분) |
| 툴 5개 이상 (Premiere, Runway, Whisper, Suno, Canva) | 단일 플랫폼 |
| 전문 지식 필요 (타임라인, 코덱, 렌더링 설정) | 프롬프트만으로 시작 가능 |
| AI 영상 생성 + 편집 툴 분리 | 생성 → 편집 → 내보내기 원스톱 |

---

### 1-3. 단계별 목표 (OKR 프레임)

#### Phase 1 — MVP 검증 (4~6주)
- **목표**: "텍스트→영상" 핵심 루프 작동하는 웹 프로토타입
- KR1: 텍스트 프롬프트 입력 → 영상 생성 → 다운로드 E2E 플로우 완성
- KR2: 이미지→영상 변환 (모션 추가) 기능 작동
- KR3: Whisper 자동 자막 생성 + SRT 내보내기
- 성공 기준: 내부 사용자 3명이 5회 이상 반복 사용

#### Phase 2 — 편집 고도화 (8~12주)
- **목표**: 타임라인 에디터로 전문 편집 워크플로우 지원
- KR1: 트랙 기반 타임라인 에디터 (영상/오디오/자막 레이어) 출시
- KR2: 씬 감지 자동 컷 기능 (FFmpeg + AI 분석)
- KR3: BGM 자동 추천 및 삽입 (Suno/ElevenLabs API)
- KR4: 실시간 프리뷰 (WebGL 렌더)

#### Phase 3 — 데스크톱 확장 (16~20주)
- **목표**: Electron 기반 데스크톱 앱 + 로컬 AI 탑재
- KR1: Electron 래핑 + 네이티브 파일 시스템 접근
- KR2: 로컬 모델 통합 (Whisper.cpp, Stable Diffusion 로컬)
- KR3: 오프라인 편집 + 로컬 렌더링 지원
- KR4: 클라우드-로컬 동기화 (작업 이어하기)

#### Phase 4 — 생태계 확장
- **목표**: 협업 + 마켓플레이스로 플랫폼화
- 팀 협업 (코멘트, 공유, 권한 관리)
- 템플릿 마켓 (제작자 수익화)
- 영상 스타일 전이 (레퍼런스 영상 스타일 복제)

## 2. 기능 설계 (Feature Spec)

### 2-1. MVP 핵심 기능

#### A. 텍스트 → 영상 생성 (Text-to-Video)

**사용자 플로우**
```
[프롬프트 입력창]
   ↓ 텍스트 입력 + 스타일 선택 (시네마틱/애니메이션/다큐 등)
[AI 생성 요청]
   ↓ 외부 API 호출 (Runway ML / Stability AI / Kling AI)
[생성 진행 표시]
   ↓ 진행률 바 + 예상 시간 표시
[미리보기]
   ↓ 브라우저 내 재생
[다운로드 / 편집으로 이동]
```

**세부 기능**
- 프롬프트 도우미: 영상 설명 → 최적화된 프롬프트 자동 생성 (GPT-4o 활용)
- 스타일 프리셋: 시네마틱, 애니메이션, 브이로그, 광고, 뉴스 5가지
- 해상도 옵션: 720p / 1080p / 세로형(9:16) / 가로형(16:9)
- 생성 이력: 최근 생성 영상 20개 보관

#### B. 이미지 → 영상 변환 (Image-to-Video)

**사용자 플로우**
```
[이미지 업로드]
   ↓ 드래그앤드롭 또는 파일 선택 (JPG/PNG/WebP)
[모션 스타일 선택]
   ↓ Zoom In / Zoom Out / Pan Left / Pan Right / Cinematic Float
[영상 길이 설정]
   ↓ 3초 / 5초 / 10초 선택
[생성 & 다운로드]
```

**AI 모델 연동**: Stability AI Image-to-Video, Runway Gen-3 Alpha, Kling AI 2.0

#### C. AI 편집 보조 (AI Editing Tools)

| 기능 | 설명 | 사용 모델/기술 |
|------|------|--------------|
| 자동 자막 생성 | 영상 업로드 → 음성 인식 → 자막 오버레이 | OpenAI Whisper API |
| 씬 감지 자동 컷 | 영상 장면 전환 자동 감지 → 클립 분할 | FFmpeg scdetect filter |
| 배경 제거 | 인물/사물 배경 자동 제거 (누끼) | Remove.bg API / rembg 로컬 |
| BGM 추천 | 영상 분위기 분석 → 어울리는 배경음악 매칭 | Suno API / ElevenLabs |
| 자막 스타일 편집 | 폰트/색상/위치/크기 편집 + SRT 내보내기 | 자체 Canvas 렌더링 |

## 5. 기술 스택 상세

### 5-1. 프론트엔드

| 기술 | 선택 이유 | 대안 |
|------|---------|------|
| React 18 + TypeScript | 컴포넌트 재사용성, 생태계 풍부, 타입 안전성 | Vue 3 (학습 곡선 낮지만 생태계 작음) |
| Zustand | 경량 상태관리, 보일러플레이트 최소, 러닝커브 낮음 | Redux (과도한 복잡도), Jotai (비슷하나 Zustand 더 성숙) |
| Vite | HMR 속도, 번들 최적화, 설정 간단 | CRA (느린 빌드), Webpack (설정 복잡) |
| WebGL (Three.js / PixiJS) | 실시간 영상 컴포지팅, 60fps 타임라인 프리뷰 | Canvas 2D API (성능 한계) |
| React DnD | 타임라인 드래그앤드롭, 클립 배치 | Sortable.js (타임라인 특화 기능 부족) |
| TailwindCSS | 빠른 UI 구축, 일관된 디자인 토큰 | Styled-components (런타임 오버헤드) |

### 5-2. 백엔드

| 기술 | 선택 이유 | 대안 |
|------|---------|------|
| Node.js + Express | AI API 비동기 처리에 최적, 프론트와 언어 통일 | Python FastAPI (ML 친화적이나 언어 분리) |
| BullMQ + Redis | 영상 생성 작업 큐 관리, 재시도/실패 처리, 우선순위 | RabbitMQ (더 복잡), SQS (AWS 의존) |
| FFmpeg | 업계 표준 영상 처리, 오픈소스, 모든 포맷 지원 | 상용 솔루션 (비용 높음) |
| Socket.io | 실시간 생성 진행률 푸시, 양방향 통신 | SSE (단방향, 심플하지만 제한적) |
| Multer | 파일 업로드 미들웨어, S3/Cloud Storage 연동 용이 | — |

## 7. 리스크 & 과제

### 7-1. 기술 리스크

| 리스크 | 심각도 | 발생 확률 | 대응 방안 |
|-------|-------|---------|---------|
| AI API 응답 지연 (30초~5분) | 높음 | 높음 | BullMQ 비동기 큐 + 진행률 WebSocket 푸시로 UX 보완 |
| 브라우저 내 영상 처리 성능 한계 | 중간 | 중간 | 경량 편집은 WebGL, 고품질 렌더는 서버 FFmpeg로 분리 |
| Electron 앱 파일 용량 (200MB+) | 낮음 | 높음 | 로컬 AI 모델은 선택적 다운로드, 핵심 기능은 클라우드 |
| AI API 서비스 중단/정책 변경 | 높음 | 중간 | 멀티 프로바이더 폴백 구조 (Runway → Kling → Stability) |

### 7-2. 비즈니스 리스크

| 리스크 | 대응 방안 |
|-------|---------|
| AI 생성 영상 저작권 이슈 | 각 API 이용약관 검토, 생성물 상업 이용 조건 사용자에게 고지 |
| API 비용 급증 | 사용량 모니터링 대시보드 + 월별 예산 캡 설정 + 사용자별 크레딧 시스템 |
| 생성 품질 편차 | 프리뷰 먼저 확인 후 고품질 렌더링 과금 구조로 비용 낭비 방지 |
| 경쟁사 (Runway, Pika, CapCut AI) | 내부 워크플로우 최적화에 집중, 외부 경쟁보다 팀 생산성 향상에 초점 |

### 7-3. 브라우저 제약사항

- **WebCodecs API**: 최신 Chrome/Edge에서만 지원 → 구 버전은 FFmpeg.wasm으로 폴백
- **메모리 한계**: 대용량 영상(4K) 브라우저 편집 불가 → 서버 렌더링으로 우회
- **CORS**: AI API 직접 호출 불가 → 반드시 백엔드 프록시 경유
